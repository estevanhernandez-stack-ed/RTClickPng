## GitHub release body (markdown)

> The first public preview. Unsigned MSIX for Windows 11 developers and early
> testers while the Microsoft Store submission is in flight. Once Store
> certification passes, the Store listing becomes the primary install path
> and `v0.1.0-preview` here gets retired in favor of `v0.1.0` proper.
>
> ### Install
>
> From an elevated PowerShell, with Windows 11 Developer Mode enabled
> (Settings → System → For developers):
>
> ```powershell
> Add-AppxPackage .\RightClickPNG_0.1.0_x64.msix -AllowUnsigned
> ```
>
> That's the whole install — no installer GUI, no reboot. Uninstall via
> Settings → Apps → Installed apps, or:
>
> ```powershell
> Get-AppxPackage 626LabsLLC.RightClicktoPNG | Remove-AppxPackage
> ```
>
> ### What this release does
>
> - **Right-click → Convert to PNG** on `.webp`, `.avif`, `.heic`, `.heif`,
>   `.jpg`, `.jpeg`, `.bmp`, `.tif`, `.tiff`, `.gif` (first frame). Writes
>   a `.png` next to the source; original untouched.
> - **Right-click → Copy as PNG** — PNG bytes straight to the clipboard in
>   three formats. Pastes cleanly into Teams, Slack, Discord, Figma,
>   Photoshop, Outlook, Word, PowerPoint, OneNote, Paint, Snipping Tool.
> - **JPEG variants** (Convert / Copy as JPEG) hidden behind a settings
>   toggle — off by default, on via Start menu → *Right Click PNG* → toggle.
> - **Batch processing** — multi-select files, right-click, one action runs
>   against the whole selection with a single summary toast.
> - **Confirm-before-overwrite** — prompts before replacing an existing
>   target file. Toggle off if you prefer silent overwrites.
> - **Branded Settings window** with a dark title bar that matches the app
>   body, launched from the Start tile or `Right Click PNG → settings…`.
>
> ### What this release doesn't do
>
> - **No network calls.** No telemetry, no crash reports, no update checks,
>   no analytics. Install offline, run offline, stays offline. Full privacy
>   policy in [PRIVACY.md](https://github.com/estevanhernandez-stack-ed/RTClickPng/blob/main/PRIVACY.md).
> - **No account.** Nothing to sign up for.
> - **No ARM64 build yet.** x64 only this release; runs under emulation on
>   ARM64 Windows.
>
> ### Verify the download
>
> SHA256 of the `.msix` is in the attached `.sha256` file.
>
> Once installed, confirm the identity matches Partner Center:
>

> ```
>

> - PackageFamilyName: `626LabsLLC.RightClicktoPNG_wz1chhb2h2v4a`
> - SignatureKind: `None` (unsigned preview — Store-installed copies will show `Store`)
>

## Microsoft Store — Product description (≤ 10,000 chars)

```
Some images just don't download and paste. You save a .webp from the web, try to drop
it into Teams or Figma, and the app rejects it. You save a .heic from
your phone, and your image editor refuses to open it. The workaround is
always the same: find an online converter, upload the file, download a
PNG, move it to where you actually needed it.

Right Click PNG replaces that detour with a right-click.

• Right-click any .webp, .avif, .heic, .heif, .jpg, .jpeg, .bmp, .tif,
  .tiff, or .gif file in File Explorer.
• Pick "Convert to PNG" to save a PNG next to the original.
• Pick "Copy as PNG" to send PNG bytes to the clipboard, ready to paste
  into Teams, Slack, Discord, Figma, Photoshop, Outlook, Word,
  PowerPoint, OneNote, Paint, or Snipping Tool.
• Done.

JPEG output is also available — both Convert to JPEG and Copy as JPEG —
hidden behind a settings toggle for anyone who needs it.

Multi-select files and the right-click action runs against the whole
selection in one shot, with a single summary toast when the batch
completes.

The original file is never modified, moved, or deleted. ICC color
profiles are preserved when the source has one. EXIF orientation is
baked into the pixel data before the metadata is stripped, so rotated
photos stay rotated.

OFFLINE BY DESIGN

No network calls of any kind. No telemetry, no crash reports, no update
checks, no analytics, no phone-home. No account, no sign-in, no cloud
sync. Install it, use it, that's the whole surface. Disconnect from the
internet and every feature still works.

See the full privacy policy linked below — it's two paragraphs because
there's almost nothing to say.

BUILT WITH NATIVE WINDOWS TECH

The engine is a native-AOT compiled converter — no .NET runtime needed
beyond what ships in the package, no interpreter warm-up. The shell
extension is a Win32 COM server that Windows File Explorer hosts directly,
the same way it hosts Windows' own right-click verbs. The settings window
is a branded WPF app with a dark title bar that matches the app body.

Decoders are statically linked from battle-tested open-source libraries:
libpng, libjpeg-turbo, libwebp, libavif, libheif, libspng, libtiff. Full
attribution in the About page.

SYSTEM REQUIREMENTS

Windows 11 (build 22621 or later), x64. ARM64 native build deferred to a
future release — x64 runs under emulation on ARM64 Windows with a
performance hit for now.

KNOWN LIMITATIONS

• Animated GIFs convert as first-frame only; no animated-PNG output.
• Very large source files (multi-hundred MB) will consume corresponding
  memory during decode. No preset cap.

SUPPORT

Bug reports and feature requests:
https://github.com/estevanhernandez-stack-ed/RTClickPng/issues

Security-sensitive reports: estevan.hernandez@gmail.com with subject
line starting [RTClickPng security].

LICENSE

MIT. Source available on GitHub.
```

## Partner Center — Notes to reviewers (initial submission)

Copy this block into the "Notes for certification" field on the submission page.
Tuned to the specific surface reviewers examine for shell-extension apps with
`runFullTrust`. Roughly ~2,400 chars; Partner Center's field accepts up to ~10k.

```
Thanks for reviewing Right Click to PNG.

ARCHITECTURE

The package is three processes under one MSIX identity:

1. ShellExtension.dll — Win32 COM in-proc server, registered via the
   manifest's com:SurrogateServer.  Hosted by dllhost.exe under our
   package identity when Explorer needs our context-menu verbs.
2. RTClickPng.Engine.exe — Native-AOT .NET 9 console converter.  Launched
   per-verb by the shell extension via CreateProcessW with a 60s cap.
   Decodes source formats via statically-linked libpng / libjpeg-turbo /
   libwebp / libavif / libheif / libspng / libtiff; encodes to PNG or JPEG.
3. RTClickPng.Settings.exe — WPF settings window reached from the Start
   tile or from the 'Right Click PNG -> settings...' context-menu verb.
   Reads/writes a single settings.json in ApplicationData.LocalState.

No network I/O anywhere in the three binaries.  We verified this with a
source-level audit (no HTTP/TLS client code linked) and by install-running
under a Windows Firewall block rule on the package family name for the
duration of a full test pass — every feature still worked.

WHAT WE TESTED END-TO-END BEFORE SUBMITTING

- Install via MSIX (Add-AppxPackage) on a clean Windows 11 23H2 VM.
- Right-click context menus on every supported source extension:
  .webp, .avif, .heic, .heif, .jpg, .jpeg, .png, .bmp, .tif, .tiff, .gif.
- Convert flows: output files land next to source, originals untouched,
  ICC profile preserved where present, EXIF orientation baked into pixel
  data before metadata strip.
- Copy flows: PNG bytes on the clipboard paste cleanly into Teams, Slack,
  Discord, Figma, Photoshop, Outlook, Word, PowerPoint, OneNote, Paint,
  Snipping Tool — verified each.
- Batch mode: multi-selection of 47 mixed-format files converts to a
  summary toast; failures isolated per-file.
- Settings toggle: disabling JPEG variants hides the JPEG verbs on next
  right-click without a restart or reboot.
- Overwrite-confirm prompt fires when target exists and toggle is on;
  silent overwrite when toggle is off.
- Settings window launches from the Start tile AND from the
  'Right Click PNG -> settings...' verb.  Both routes show the WPF UI
  with a dark title bar matching the body (DwmSetWindowAttribute with
  DWMWA_USE_IMMERSIVE_DARK_MODE + caption/border/text color attributes).
- Uninstall (Remove-AppxPackage) leaves no registry keys, no scheduled
  tasks, no leftover files outside the standard ApplicationData scope
  (which Windows handles).

RESTRICTED CAPABILITY JUSTIFICATION (runFullTrust)

Right Click to PNG is a Win32 shell extension (IExplorerCommand /
IContextMenu) hosted in dllhost.exe under our package identity, plus a
native-AOT engine that decodes third-party image formats and writes files
next to the source.  Both require Win32-style filesystem access and
in-process COM registration that the sandboxed packaged-app APIs don't
expose.  No network, no elevated privileges, no broadFileSystemAccess.

CONTEXT-MENU POLICY (10.8)

Verbs show only when FileFilter::ShouldShow* returns true for the
current selection — never on non-image files.  JPEG variants default OFF
and require explicit user opt-in via settings.  Single grouped
'Right Click PNG ->' parent label, not sprinkled top-level entries.

60-SECOND REVIEW PATH

1. Install MSIX.  Start menu should show 'Right Click PNG' tile.
2. Save a .webp from a browser to your Desktop.
3. Right-click it -> Right Click PNG -> Convert to PNG.  Toast confirms,
   .png appears next to the .webp.
4. Right-click the .webp -> Copy as PNG.  Paste into Paint.  Image lands.
5. Click the Start tile -> Settings window opens with two toggles.
6. Flip 'Show JPEG variants' on.  Close.  Right-click the .webp again —
   Convert to JPEG and Copy as JPEG now appear in the submenu.

HONEST DISCLOSURES

- This is v0.1.1.0, our first public release.  4-part MSIX Version advanced
  past 0.1.0.x while we iterated on the DisplayName reservation and version-
  uniqueness constraints; the 0.1.1.0 package is the first to clear both.
- GIF sources convert first-frame only; no animated PNG output.  Documented
  in the listing.
- x64 only; ARM64 native build deferred to a later release.  x64 runs under
  emulation on ARM64 Windows with a performance hit, documented as a known
  limitation.
- Source and build pipeline are public at github.com/estevanhernandez-stack-ed/RTClickPng
  if a reviewer wants to cross-check any of the above against actual code.

Thanks for the careful review.  Happy to answer follow-ups at
estevan.hernandez@gmail.com.
```
